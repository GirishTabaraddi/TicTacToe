The task is to develop a console based version of the board game Tic\+Tac\+Toe


\begin{DoxyItemize}
\item The game consists of a 3x3/4x4/5x5 board.
\item On every position, the players place a “x” (player 1) or a “o” (Player 2) stone.
\item The first player, who has 3 stones in a row (horizontally, vertically or diagonally) wins the game.
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md11}{}\doxysubsubsection{\texorpdfstring{Friend Class}{Friend Class}}\label{md__r_e_a_d_m_e_autotoc_md11}
A {\bfseries{Friend Class}} is a class that is granted special access to the {\bfseries{private}} and {\bfseries{protected}} members of another class. Under normal object-\/oriented rules, private data is hidden from outside classes. However, if Class A declares Class B as a {\ttfamily friend}, Class B can access Class A\textquotesingle{}s private variables and functions.

{\bfseries{Key Characteristics\+:}}
\begin{DoxyItemize}
\item {\bfseries{Not Symmetric\+:}} If A is a friend of B, B is not automatically a friend of A.
\item {\bfseries{Not Transitive\+:}} If A is a friend of B, and B is a friend of C, A is {\itshape not} automatically a friend of C.
\item {\bfseries{Not Inherited\+:}} Friendships are not passed down to derived classes.
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md12}{}\doxysubsubsection{\texorpdfstring{Static Attribute (\+Static Data Member)}{Static Attribute (Static Data Member)}}\label{md__r_e_a_d_m_e_autotoc_md12}
A {\bfseries{Static Attribute}} is a variable that belongs to the {\bfseries{class itself}}, rather than to any specific object (instance) of that class. No matter how many objects of the class you create, there is only {\bfseries{one copy}} of the static variable in memory. All objects share this single variable.

{\bfseries{Key Rules\+:}}
\begin{DoxyItemize}
\item {\bfseries{Shared Memory\+:}} Changing the value in one object changes it for all objects.
\item {\bfseries{Initialization\+:}} Static members must be defined and initialized {\bfseries{outside}} the class definition (usually in the {\ttfamily .cpp} file) to allocate memory.
\item {\bfseries{Lifetime\+:}} It exists for the entire duration of the program, initialized before {\ttfamily main()} runs.
\end{DoxyItemize}

\hypertarget{md__r_e_a_d_m_e_autotoc_md13}{}\doxysubsubsection{\texorpdfstring{Static Method}{Static Method}}\label{md__r_e_a_d_m_e_autotoc_md13}
A {\bfseries{Static Method}} is a function that belongs to the class rather than an instance. It can be called without creating an object of the class.

{\bfseries{Key Rules\+:}}
\begin{DoxyItemize}
\item {\bfseries{No {\ttfamily this} Pointer\+:}} Since it is not attached to an object, it does not have a {\ttfamily this} pointer.
\item {\bfseries{Limited Access\+:}} A static method can {\bfseries{only}} access {\bfseries{Static Attributes}} or call other {\bfseries{Static Methods}}. It cannot access non-\/static (regular) member variables because it doesn\textquotesingle{}t know which object\textquotesingle{}s data to look at.
\item {\bfseries{Invocation\+:}} Called using the scope resolution operator {\ttfamily Class\+Name\+::\+Method\+Name()}.
\end{DoxyItemize}

 \hypertarget{md__r_e_a_d_m_e_autotoc_md14}{}\doxysubsubsection{\texorpdfstring{Throwing and Catching Exceptions}{Throwing and Catching Exceptions}}\label{md__r_e_a_d_m_e_autotoc_md14}
Exception handling provides a way to transfer control from one part of a program to another when an error or "{}exceptional"{} situation occurs. It separates error-\/handling code from normal program logic.

{\bfseries{Keywords\+:}}
\begin{DoxyItemize}
\item \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}{\ttfamily try}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+: A block of code where exceptions might occur.
\item \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}{\ttfamily throw}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+: Used to signal that an error has occurred. You can throw integers, strings, or objects (like {\ttfamily std\+::exception}).
\item \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}{\ttfamily catch}\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+: A block of code that handles the specific exception thrown.
\end{DoxyItemize}

{\bfseries{Key Rules\+:}}
\begin{DoxyItemize}
\item {\bfseries{Stack Unwinding\+:}} When an exception is thrown, the program leaves the current scope immediately. Local objects are destroyed (destructors are called) as the program looks for a {\ttfamily catch} block.
\item {\bfseries{Handlers\+:}} You can have multiple {\ttfamily catch} blocks for different types of errors. {\ttfamily catch(...)} catches anything.
\end{DoxyItemize}
